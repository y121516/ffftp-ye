# FFFTP 仕様ドキュメント

## 1. コマンドラインオプション

FFFTPは、起動時に様々なコマンドラインオプションを受け付け、その動作を制御します。これらのオプションは、特に自動テストのシナリオにおいて、FFFTPの挙動を外部から細かく制御するために不可欠です。

### 1.1. オプション一覧

以下の表に、FFFTPがサポートする主要なコマンドラインオプションを示します。ユーザー提供のサイト [^1] およびソースコード (`src/main.c` の `AnalyzeComLine` 関数) の解析に基づいています。

| 短縮形 | 長形式 | 引数 | 説明 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| `-s` | `--set` | `<ホスト設定名>` | FFFTPに保存されているホスト設定名を指定して接続します。 | 自動接続テストの基本 |
| | `<URL>` | `<URL>` | FTPサーバーのURLを直接指定して接続します。書式: `[ftp://][[userID][:password]@]hostname[:][port#][filename]` | 一時的な接続やURLベースのテストに有用 |
| `-a` | `--ascii` | なし | ASCIIモードでファイルを転送します（テキストファイル）。 | 転送モードの固定 |
| `-b` | `--binary` | なし | バイナリモードでファイルを転送します（バイナリデータ）。 | 転送モードの固定 |
| `-x` | `--auto` | なし | 自動判定モードでファイルを転送します（ファイル種別に応じて選択）。 | 転送モードの固定 |
| `-eu` | `--euc` | なし | ホスト側の文字コードとしてEUCを使用します。 | 文字コード変換テスト |
| `-ji` | `--jis` | なし | ホスト側の文字コードとしてJISを使用します。 | 文字コード変換テスト |
| `-sj` | `--sjis` | なし | ホスト側の文字コードとしてSJISを使用します。 | 文字コード変換テスト |
| `-u8` | `--utf8` | なし | ホスト側の文字コードとしてUTF-8 (BOMなし) を使用します。 | 文字コード変換テスト |
| `-8b` | `--utf8bom` | なし | ホスト側の文字コードとしてUTF-8 (BOMあり) を使用します。 | 文字コード変換テスト |
| `-eun` | `--eucname` | なし | ホスト側のファイル名の文字コードとしてEUCを使用します。 | ファイル名文字コード変換テスト |
| `-jin` | `--jisname` | なし | ホスト側のファイル名の文字コードとしてJISを使用します。 | ファイル名文字コード変換テスト |
| `-sjn` | `--sjisname` | なし | ホスト側のファイル名の文字コードとしてSJISを使用します。 | ファイル名文字コード変換テスト |
| `-u8n` | `--utf8name` | なし | ホスト側のファイル名の文字コードとしてUTF-8を使用します。 | ファイル名文字コード変換テスト |
| `-m` | `--mirror` | なし | 接続後、ミラーリングアップロードを自動的に開始します。 | 自動ミラーリングテスト |
| `-d` | `--mirrordown` | なし | 接続後、ミラーリングダウンロードを自動的に開始します。 | 自動ミラーリングテスト |
| `-f` | `--force` | なし | ミラーリング時に開始確認を行わず実行します。 | **自動テストに必須** |
| `-q` | `--quit` | なし | 転送完了後、FFFTPを自動終了します。 | **自動テストに必須** |
| `--saveoff` | なし | 設定変更時、設定を保存しないようにします。 | テスト実行による設定変更の防止 |
| `--saveon` | なし | 設定変更時に設定を保存するように戻します。 | |
| `-z` | `--mpasswd` | `<password>` | マスターパスワードを指定します。 | 暗号化された設定へのアクセス |
| `-n` | `--ini` | `<INIファイル>` | 指定したINIファイルの設定で起動します。フルパスで指定する必要があります。 | **テスト環境の分離に必須** |
| `-h` | `--help` | なし | HTMLヘルプを開きます。 | |

### 1.2. コマンドライン引数の優先順位

FFFTPは、起動時に以下の順序でコマンドライン引数を処理します。

1.  `-n <INIファイル>`: このオプションが指定された場合、FFFTPは指定されたINIファイルを読み込みます。指定がない場合は、`FFFTP.exe` と同じフォルダの `FFFTP.ini` がデフォルトで利用されます。
2.  `-z <マスターパスワード>`: 暗号化された設定（ホストパスワードなど）を復号するために使用されます。
3.  接続指定 (`-s` または `<URL>`) およびその他の操作オプション (`-m`, `-d`, `-a`, `-b`, `-x`, 文字コード関連など) が処理され、FFFTPの動作が決定されます。

## 2. FFFTP.ini ファイル仕様

`FFFTP.ini` ファイルは、FFFTPの各種設定を保存するために使用されます。このファイルは、FFFTPの挙動を細かく制御するための中心的な役割を果たします。特に、コマンドラインオプションでは指定できない詳細な設定（例: PASVモードの有効/無効、特定のホストタイプ設定など）を制御するために重要です。

### 2.1. ファイルの場所と読み込み

*   デフォルトでは、`FFFTP.exe` と同じディレクトリにある `FFFTP.ini` が読み込まれます。
*   コマンドラインオプション `-n <INIファイル>` を使用することで、任意のINIファイルを指定できます。

### 2.2. 主要なセクションと設定項目

`FFFTP.ini` は、主に以下のセクションで構成されます。各セクションには、FFFTPの挙動を定義するキーと値のペアが含まれます。以下は、ゴールデンマスターテストにおいて特に重要となる項目の一部です。ソースコード (`src/registry.c` の `LoadRegistry` 関数) の解析に基づいています。

#### `[FFFTP\Options]` セクション (アプリケーション共通設定)

| キー | 型 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| `Debug` | `int` | デバッグコンソールの表示 (0:無効, 1:有効)。 | ログ出力の有効化に影響する可能性あり |
| `Log` | `int` | ログのファイル出力 (0:無効, 1:有効)。 | **テストログ取得に必須** |
| `TransMode` | `int` | デフォルトの転送モード (0:ASCII, 1:Binary, 2:Auto)。 | コマンドラインオプション `-a`, `-b`, `-x` と関連 |
| `AsciiFile` | `string` | ASCIIモードで転送するファイルの拡張子リスト (例: `txt;html;css;`)。 | `TransMode=2` (Auto) の挙動に影響 |
| `SaveWinPos` | `int` | ウィンドウ位置とサイズを保存するか (0:しない, 1:する)。 | テスト実行によるGUI状態の変更を防止 |
| `RemoveOldLog` | `int` | 古い処理内容を消去するか (0:しない, 1:する)。 | ログファイル管理 |
| `MasterPassword` | `string` | マスターパスワードのハッシュ値。 | 暗号化された設定の復号に必要 |
| `EncryptAll` | `int` | 全設定を暗号化するか (0:しない, 1:する)。 | |

#### `[FFFTP\HostX]` セクション (ホスト個別設定)

`X` はホストのインデックス（0から始まる整数）です。各ホスト設定は、このセクションの下に保存されます。

| キー | 型 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| `HostName` | `string` | ホスト名。 | |
| `HostAdrs` | `string` | ホストアドレス (IPアドレスまたはドメイン名)。 | |
| `UserName` | `string` | ユーザー名。 | |
| `Password` | `string` | パスワード (暗号化されて保存)。 | |
| `Port` | `int` | ポート番号。 | デフォルトは21 |
| `Kanji` | `int` | ファイル内容の漢字コード (0:SJIS, 1:EUC, 2:JIS, 3:UTF-8(BOMなし), 4:UTF-8(BOMあり))。 | コマンドラインオプション `-u8` 等と関連 |
| `NameKanji` | `int` | ファイル名の漢字コード (0:SJIS, 1:EUC, 2:JIS, 3:UTF-8)。 | コマンドラインオプション `-u8n` 等と関連 |
| `Pasv` | `int` | PASVモードを使用するか (0:しない, 1:する)。 | 通信シーケンスに影響 |
| `Type` | `int` | ホストタイプ (0:UNIX, 1:Windows, 2:VMS, 3:NetWare, 4:その他)。 | `LIST` コマンドの解釈に影響 |
| `UseFTPES` | `int` | FTPES (Explicit SSL/TLS) を使用するか (0:しない, 1:する)。 | 暗号化通信テスト |
| `UseFTPIS` | `int` | FTPIS (Implicit SSL/TLS) を使用するか (0:しない, 1:する)。 | 暗号化通信テスト |
| `UseSFTP` | `int` | SFTPを使用するか (0:しない, 1:する)。 | 暗号化通信テスト |
| `LocalDir` | `string` | ローカル側の初期ディレクトリ。 | ミラーリングテストの基準 |
| `RemoteDir` | `string` | リモート側の初期ディレクトリ。 | ミラーリングテストの基準 |
| `Init` | `string` | 接続後に実行する初期コマンド。 | 特定のFTPコマンドテスト |
| `Noop` | `int` | NOOPコマンドの送信間隔 (秒)。 | 接続維持テスト |
| `TransferErrorMode` | `int` | 転送エラー時の動作 (0:中断, 1:スキップ, 2:再試行)。 | エラーハンドリングテスト |

### 2.3. INIファイルの生成とテストへの応用

上記のキーと値を組み合わせることで、特定のテストシナリオに合わせた `ffftp.ini` ファイルをスクリプトで生成することが可能です。例えば、UTF-8のファイル名でミラーリングアップロードを行うテストの場合、`[FFFTP\Host0]` セクションの `NameKanji` を `3` に設定し、`LocalDir` と `RemoteDir` を適切に指定したINIファイルを生成します。

## 3. テストシナリオと設定項目のマッピング

FFFTP 2.0の挙動を完全に再現し、近代化されたバージョンとの互換性を検証するためには、網羅的なテストシナリオと、それに対応するFFFTPの設定を定義することが不可欠です。ここでは、主要なテストカテゴリと、それらを制御するための設定項目をマッピングします。

### 3.1. 主要なテストカテゴリ

| テストカテゴリ | 目的 | 関連するINI設定項目 | 関連するコマンドラインオプション |
| :--- | :--- | :--- | :--- |
| **基本接続** | FTPサーバーへの正常な接続、認証の確認。 | `HostName`, `HostAdrs`, `UserName`, `Password`, `Port` | `-s`, `<URL>` |
| **ファイル転送（アップロード/ダウンロード）** | 各種ファイル（テキスト、バイナリ、サイズ違い）の正確な転送。 | `TransMode`, `AsciiFile` | `-a`, `-b`, `-x` |
| **文字コード変換** | ファイル内容およびファイル名の文字コード変換の正確性。 | `Kanji`, `NameKanji` | `-eu`, `-ji`, `-sj`, `-u8`, `-8b`, `-eun`, `-jin`, `-sjn`, `-u8n` |
| **ミラーリング** | ディレクトリ構造を維持したままのアップロード/ダウンロード。 | `LocalDir`, `RemoteDir` | `-m`, `-d`, `-f`, `-q` |
| **PASV/PORTモード** | データコネクション確立方式の確認。 | `Pasv` | なし |
| **FTPS/SFTP接続** | 暗号化された通信の確立とデータ転送。 | `UseFTPES`, `UseFTPIS`, `UseSFTP` | なし |
| **特殊ファイル名** | 日本語、スペース、特殊記号を含むファイル名の処理。 | `NameKanji` | `-eun`, `-jin`, `-sjn`, `-u8n` |
| **エラーハンドリング** | 接続失敗、転送中断、タイムアウト時の挙動。 | `Noop`, `TransferErrorMode` | なし |
| **初期コマンド** | 接続後に自動実行されるFTPコマンドの確認。 | `Init` | なし |

### 3.2. テストシナリオの例

各テストカテゴリは、さらに具体的なシナリオに分解されます。例えば、「文字コード変換」のテストカテゴリには、以下のようなシナリオが考えられます。

*   **シナリオ1: UTF-8ファイル名でのミラーリングアップロード**
    *   **目的**: ローカルのUTF-8ファイル名（日本語含む）が、リモートサーバーで正しく扱われ、FFFTPのログにも正確に記録されることを確認する。
    *   **INI設定**: `[FFFTP\HostX]` セクションで `NameKanji=3` (UTF-8) を設定。
    *   **コマンドライン**: `ffftp.exe -s "UTF8_TestHost" -m -f -q -u8n`
    *   **テストデータ**: 日本語ファイル名（例: `テストファイル.txt`）を含むローカルディレクトリ。

*   **シナリオ2: SJISファイル内容のASCIIモードダウンロード**
    *   **目的**: SJISで記述されたテキストファイルが、ASCIIモードでダウンロードされた際に改行コード変換が正しく行われることを確認する。
    *   **INI設定**: `[FFFTP\HostX]` セクションで `Kanji=0` (SJIS) を設定。
    *   **コマンドライン**: `ffftp.exe -s "SJIS_TextHost" -d -f -q -a`
    *   **テストデータ**: SJISエンコーディングのテキストファイル（例: `sample.txt`）が配置されたリモートディレクトリ。

## 4. ゴールデンマスターテスト実施手順（非侵襲アプローチ）

ソースコードに一切変更を加えず、FFFTP 2.0の既存機能と外部スクリプトを組み合わせてゴールデンマスターテストを実施します。これにより、テスト基盤の導入自体による「2.0の挙動」への影響を排除します。

### 4.1. 前提条件

*   **FFFTP 2.0バイナリ**: 比較対象となるオリジナルのFFFTP 2.0実行ファイル (`ffftp.exe`) が用意されていること。
*   **テスト用FTPサーバー**: 挙動が固定されたローカルFTPサーバー（例: Dockerコンテナで動作するvsftpd）が用意されており、テストデータが配置されていること。
*   **テスト用データセット**: 各シナリオに対応するローカル/リモートのファイル群（文字コード、ファイルサイズ、種類など）が準備されていること。

### 4.2. テスト実施フロー

1.  **テスト用INIファイルの生成**: 各テストシナリオに対応する `ffftp.ini` ファイルをスクリプトで生成します。この際、`[FFFTP\Options]` セクションの `Log=1` を設定し、タスクログがファイルに出力されるようにします。
2.  **FFFTP 2.0の実行**: 生成したINIファイルと適切なコマンドライン引数を用いて、FFFTP 2.0を起動します。例: `ffftp.exe -n "C:\test\scenario1.ini" -s "TestHost" -m -f -q`
3.  **ログの取得**: FFFTPの実行が完了した後、出力されたタスクログファイル（通常 `ffftp.log` または設定されたパス）を回収します。
4.  **ログの正規化**: 取得したログファイルから、実行ごとに変動する要素（タイムスタンプ、ソケットID、プロセスIDなど）を正規表現を用いて一貫したプレースホルダー（例: `[TIMESTAMP]`, `[SOCKET_ID]`）に置換します。これにより、本質的なプロトコル挙動のみを比較対象とします。
5.  **ゴールデンマスターの確立**: FFFTP 2.0で実行し、正規化されたログを「ゴールデンマスターログ」として保存します。各シナリオに対して1つのゴールデンマスターログを作成します。
6.  **近代化版FFFTPの実行**: 開発中の近代化版FFFTPバイナリに対して、ステップ1〜4と全く同じ手順でテストを実行し、ログを取得・正規化します。
7.  **ログの比較**: 正規化された近代化版のログと、対応するゴールデンマスターログを `diff` コマンドや専用の比較ツールで比較します。差異がないことを確認します。

### 4.3. ログ正規化のルール例

*   **タイムスタンプ**: `YYYY/MM/DD HH:MM:SS` の形式を `[TIMESTAMP]` に置換。
*   **ソケットID**: `(S=0xXXXXXXXX)` の形式を `(S=[SOCKET_ID])` に置換。
*   **プロセスID**: ログに出力される可能性のあるプロセスIDを `[PID]` に置換。
*   **一時ファイルパス**: `C:\Users\...\temp\` のような一時パスを `[TEMP_PATH]` に置換。
