# FFFTP ゴールデンマスターテスト 詳細テストケース案

本ドキュメントは、FFFTPのコマンドラインから実行可能な大粒度ゴールデンマスターテストの詳細計画を策定するものです。PythonのFTP/SFTPサーバーライブラリ（`pyftpdlib`および`paramiko`）を活用し、FFFTPの多様な挙動を検証するためのテスト環境設計と、具体的なテストケースを網羅的に洗い出します。

## 1. テスト環境の設計

ゴールデンマスターテストの核となるのは、FFFTPの挙動を正確に再現し、比較するための安定したテスト環境です。ここでは、Python製のFTP/SFTPサーバーをカスタマイズして、様々なシナリオをシミュレートします。

### 1.1. Python FTP/SFTPサーバーの役割

*   **プロトコル挙動の制御**: FFFTPが送信するFTP/SFTPコマンドに対して、期待される応答を返す。これにより、正常系だけでなく、エラー応答や特定のプロトコルシーケンスを再現可能にします。
*   **ファイルシステムのエミュレーション**: テスト用の仮想ファイルシステムを提供し、様々なファイル（文字コード、サイズ、種類）の配置、作成、削除、変更を制御します。
*   **ログの記録**: FFFTPとの通信内容（コマンド、応答、データ転送）を詳細に記録し、ゴールデンマスターログと比較するための情報源とします。
*   **文字コードのシミュレーション**: サーバー側で異なる文字コード（SJIS, EUC-JP, UTF-8）を使用してファイル名やファイル内容を扱うことで、FFFTPの文字コード変換機能を検証します。
*   **異常系のシミュレーション**: 接続拒否、タイムアウト、転送中断、ディスクフルなどのエラー条件を意図的に発生させ、FFFTPのエラーハンドリングをテストします。

### 1.2. 採用ライブラリとカスタマイズ方針

*   **FTP/FTPSサーバー**: `pyftpdlib` を採用します。`FTPHandler` を継承したカスタムハンドラを作成し、以下の点をカスタマイズします。
    *   `encoding` 属性を動的に変更し、サーバー側のファイル名・内容の文字コードをシミュレートします。
    *   `ftp_LIST` メソッドをオーバーライドし、UNIX形式、Windows形式など、様々なディレクトリリスト形式を返せるようにします。
    *   `ftp_FEAT` メソッドをオーバーライドし、FFFTPがサーバーの機能を正しく認識するかをテストします。
    *   `on_file_received`, `on_file_sent` などのコールバックを利用して、ファイル転送イベントをフックし、ログ記録やデータ検証を行います。
    *   `DummyAuthorizer` を利用して、ユーザー認証を制御します。
*   **SFTPサーバー**: `paramiko` を採用します。`paramiko.ServerInterface` を継承したカスタムサーバーを作成し、以下の点をカスタマイズします。
    *   SSH認証の挙動を制御し、正しいユーザー名/パスワードでの接続、誤った認証情報の拒否をシミュレートします。
    *   SFTPプロトコルレベルでのファイル操作（ファイル一覧取得、アップロード、ダウンロード）をフックし、ログ記録やデータ検証を行います。
    *   ファイル名やファイル内容の文字コード処理を検証するため、`paramiko`のエンコーディング設定を調整します。

## 3. テスト自動化スクリプトの構成案

上記のテストケースを実行するために、以下のPythonスクリプト群を開発します。

1.  **`ftp_server_manager.py`**: `pyftpdlib`および`paramiko`をラップし、テストシナリオに応じてFTP/SFTPサーバーを起動・停止・設定変更するモジュール。特定の文字コード設定、`LIST`応答、エラー挙動などを動的に切り替えられるAPIを提供します。
2.  **`ini_generator.py`**: 各テストケースの要件に基づき、`ffftp.ini`ファイルを動的に生成するスクリプト。FFFTPのコマンドラインオプションとINIファイル設定のマッピングを管理します。
3.  **`ffftp_runner.py`**: FFFTPバイナリをコマンドラインから実行し、指定されたINIファイルと引数を渡し、FFFTPのプロセス終了を待機し、ログファイル（`ffftp.log`）を回収するスクリプト。
4.  **`log_normalizer.py`**: 回収したFFFTPログから、タイムスタンプ、ソケットID、プロセスID、一時ファイルパスなど、実行ごとに変動する要素を正規表現を用いて一貫したプレースホルダーに置換するスクリプト。
5.  **`golden_master_comparator.py`**: 正規化された開発中FFFTPのログと、事前に取得・正規化されたゴールデンマスターログを比較し、差異を検出するスクリプト。`diff`ユーティリティをPythonから呼び出すか、`difflib`モジュールを使用します。
6.  **`test_runner.py`**: 上記のスクリプトを統合し、各テストケースを順次実行する上位スクリプト。テストケースの定義（JSON/YAML形式など）を読み込み、サーバーの起動、INIファイルの生成、FFFTPの実行、ログの比較、結果のレポートまでを自動化します。

## 4. 今後のステップ

1.  **Python FTP/SFTPサーバーの実装**: `ftp_server_manager.py`のコア部分を実装し、基本的なFTP/SFTP通信と、文字コード、`LIST`応答のカスタマイズ機能を確立します。
2.  **INIファイル生成とFFFTP実行スクリプトの実装**: `ini_generator.py`と`ffftp_runner.py`を実装し、FFFTPの自動実行フローを確立します。
3.  **ログ正規化と比較スクリプトの実装**: `log_normalizer.py`と`golden_master_comparator.py`を実装し、ログ比較の基盤を構築します。
4.  **テストランナーの実装**: `test_runner.py`を実装し、上記コンポーネントを統合してテストスイートを構築します。
5.  **ゴールデンマスターログの取得**: FFFTP 2.0オリジナルバイナリと完成したテスト環境を用いて、各テストケースのゴールデンマスターログを取得します。
6.  **CI/CD連携**: 完成したテストスイートをGitHub ActionsなどのCI/CDパイプラインに組み込み、継続的な互換性検証を実現します。
